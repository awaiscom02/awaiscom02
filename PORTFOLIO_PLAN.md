# AI/Python Portfolio Plan

This is the order I recommend for building a strong GitHub profile for AI Engineer and Python Developer roles.

## Step 1: Profile README

Create a public GitHub repository named exactly:

```text
awaiscom02
```

Add the `README.md` file from this folder. This will show on the main GitHub profile page.

## Step 2: First 5 Projects

### 1. rag-pdf-assistant

Build first because it shows useful LLM engineering skills.

Main features:
- upload or load PDF files
- extract text
- chunk text into smaller sections
- create embeddings
- store/search chunks
- ask questions
- return answer with sources

Good for roles involving:
- LLM Engineer
- AI Engineer
- Python Developer with AI focus

### 2. fastapi-llm-backend

Build a backend API around LLM calls.

Main features:
- FastAPI routes
- `.env` config
- request validation
- basic logging
- Dockerfile
- simple tests

### 3. llm-prompt-evaluator

Build a tool for comparing prompts.

Main features:
- test multiple prompts
- save responses
- compare latency/cost/quality notes
- export results to CSV

### 4. ai-resume-job-matcher

Build a practical AI tool for hiring/job matching.

Main features:
- upload resume text
- paste job description
- compare using embeddings or TF-IDF first
- show match score
- suggest missing skills

### 5. customer-support-ai-classifier

Build a machine learning classification project.

Main features:
- clean dataset
- train classifier
- evaluate accuracy/F1
- simple prediction API or script

## Step 3: How Each Project Should Look

Each repository should include:

- `README.md`
- `requirements.txt`
- `.env.example`
- `src/` folder
- `data/sample/` folder when useful
- screenshots if it has a UI
- clear setup instructions
- short explanation of what was learned

The code should be clean and readable, but natural. It should look like a Master's CS student built it carefully, not like an over-engineered company template.
